Faz 0 — Proje İskeleti ve Altyapı

Büyüklük: S–M · Ön koşul: yok · Durum: Planlandı · Dayanak: Spec v1.1 §14

1. Amaç

Sonraki tüm fazların üzerinde koşacağı zemini kurmak: sabit sürümlü Flutter kurulumu, katmanlı proje yapısı, mimari koruma, deterministik PRNG, Türkçe metin yardımcıları ve her push'ta çalışan CI. Bu faz bittiğinde depo "boş ama sağlam" bir uygulamadır.

2. Çıktılar

  • Web + Android + iOS hedefli, derlenen Flutter projesi (repo kökünde).
  • scripts/setup.sh: tek komutla sabitlenmiş Flutter SDK kurulumu (uzak ortam ve yeni geliştirici için).
  • Katman dizinleri + mimari koruma testi (engine/generator'da Flutter importu yasak).
  • Prng (xoshiro128**) ve tr_text yardımcıları, testleriyle.
  • GitHub Actions CI: format + analiz + test + web build.

4. Görevler

IDGörevDetay ve teknik notlarKabul kriterleri
F0.1 Flutter kurulum stratejisi ve scripts/setup.sh Stable kanaldan tek bir sürüm sabitlenir (kurulum günündeki en güncel stable; sürüm scripts/flutter-version.txt dosyasına yazılır ve CI aynı dosyayı okur). Script: SDK arşivini indirir, ~/flutter'a açar, PATH talimatı basar, flutter --version ve flutter doctor doğrulaması yapar. Uzak ortamın HTTPS proxy'siyle uyumlu (CA bundle'a saygı). Not: bu ortamda SDK kurulu değil — bu görev her şeyden önce gelir. Temiz ortamda bash scripts/setup.sh sonrası flutter test koşuyor; sürüm tek dosyadan yönetiliyor.
F0.2 Flutter projesi oluşturma flutter create --org com.manasal --project-name manasal_solitaire --platforms web,android,ios . ile repo köküne. Linux/Windows/macOS masaüstü hedefleri eklenmez. pubspec.yaml: ad, açıklama (TR), sürüm 0.1.0+1. Varsayılan sayaç uygulaması temizlenir. flutter run -d web-server placeholder ekranı açıyor; flutter build web hatasız.
F0.3 Katman dizinleri ve mimari koruma lib/{engine,content,generator,persistence,app} + test/{engine,content,generator,app} oluşturulur (spec §14.6). test/architecture_test.dart: lib/engine ve lib/generator altındaki tüm .dart dosyalarını tarar; package:flutter importu ve dart:ui bulursa test düşer. Ek kural: lib/engine içinde dart:math Random kullanımı yasak (PRNG zorunlu). Koruma testi mevcut ve yeşil; bilerek eklenen ihlalle kırmızıya düştüğü gösterildi (sonra geri alındı).
F0.4 Lint ve analiz sıkılaştırma analysis_options.yaml: flutter_lints tabanı + language: strict-casts/strict-inference/strict-raw-types + ek kurallar (always_declare_return_types, prefer_final_locals, unawaited_futures, avoid_dynamic_calls). Format: dart format varsayılanı. flutter analyze sıfır uyarıyla geçiyor.
F0.5 Deterministik PRNG lib/engine/prng.dart: xoshiro128** çekirdeği + SplitMix64 tohum genişletme. API: Prng(int seed), nextUint32(), nextInt(bound) (modulo-bias'sız, rejection sampling), shuffle<T>(List<T>) (Fisher-Yates), fork(int streamId) (bağımsız alt akış — üreticide kategori/kelime/dağıtım ayrımı için). Spec §14.4 gereği platformlar arası bit-bit aynı. Sabit tohum için ilk 8 çıktının known-answer testi; aynı tohum → aynı shuffle sırası; fork akışları bağımsız (test).
F0.6 Türkçe metin yardımcıları ve ortak tipler lib/content/tr_text.dart: trLower (İ→i, I→ı dahil), trUpper, trCapitalize, trEquals (normalize karşılaştırma). Tüm içerik karşılaştırmaları bu katmandan geçer (risk R5). lib/engine/result.dart: sealed Result<T,E> = Ok | Err yardımcı tipi. "İNCİR"→"incir", "ISPANAK"→"ıspanak", "istanbul"→"İstanbul" testleri geçiyor.
F0.7 CI (GitHub Actions) .github/workflows/ci.yml: push/PR tetikli; adımlar: checkout → Flutter kurulumu (sürüm scripts/flutter-version.txt'den) → flutter pub get → dart format --set-exit-if-changed . → flutter analyze → flutter test → flutter build web --release → web build artifact yükleme. (GitHub Pages otomatik dağıtımı Faz 3'te eklenir.) Dala push'ta workflow uçtan uca yeşil; artifact indirilebilir.
F0.8 Placeholder uygulama ve tema token iskeleti lib/main.dart + lib/app/theme/tokens.dart: renk/aralık/köşe yarıçapı sabitlerinin iskeleti (değerler Faz 3'te kağıt/kitap temasıyla dolar; şimdilik nötr). Ana ekranda "Manasal Solitaire" başlığı ve sürüm etiketi. Portre kilidi mobilde etkin (web'de serbest). Uygulama tema tokenlarını kullanıyor; kod içinde ham renk sabiti yok.
F0.9 README ve geliştirme kılavuzu güncellemesi README'ye "Geliştirme" bölümü: kurulum (scripts/setup.sh), koşma (flutter run -d chrome), test, katman kuralları özeti, doküman linkleri. Yeni bir geliştirici README ile sıfırdan ortam kurabiliyor.

6. Test Planı

  • PRNG known-answer + determinizm + fork bağımsızlığı + nextInt sınır testleri (bound 1, büyük bound).
  • Türkçe normalize birim testleri (İ/ı köşe durumları, boş dize, çift boşluk).
  • Mimari koruma testi (pozitif + bilinçli ihlalle negatif doğrulama).
  • CI'ın kendisi: workflow bir push'ta uçtan uca koşturulup doğrulanır.

7. Bitti Tanımı (DoD)

  • Tüm görev kabul kriterleri karşılandı; CI yeşil (M1 kilometre taşı).
  • bash scripts/setup.sh && flutter test temiz ortamda çalışıyor.
  • README geliştirme bölümü güncel; plan dokümanı durumu "Tamamlandı" yapıldı.

8. Riskler

  • SDK indirme engeli (R6): proxy CA'sına saygılı indirme; başarısızlıkta saf Dart katmanları dart test ile sürdürülebilir, UI işleri kullanıcının yerel makinesinde doğrulanır.
  • Sürüm kayması: Flutter sürümü tek dosyada sabit; yükseltme bilinçli ve ayrı commit'le yapılır.
