Faz 1 — Oyun Motoru

Büyüklük: L · Ön koşul: Faz 0 · Durum: Planlandı · Dayanak: Spec v1.1 §5–§10, §14.3–14.5

1. Amaç

Oyunun tüm kurallarını, Flutter'a hiç bağımlı olmayan, immutable, tamamen test edilmiş saf Dart motoru olarak gerçeklemek. Bu faz bittiğinde oyun "görünmez ama oynanabilir" durumdadır: her kural, her kenar durumu testlerle kanıtlanmıştır; çözücü (Faz 2) ve UI (Faz 3) bu motoru olduğu gibi kullanır.

İlke: Kural matrisi (spec §6.2) test adlarıyla birebir eşlenir. Bir kural değişecekse önce spec değişir, sonra test, sonra kod. Motor hiçbir zaman spec'ten sapamaz.

2. Çıktılar

  • lib/engine/: modeller, hamleler, kural doğrulama, reducer, legal-move üreteci, çıkmaz tespiti, oturum (undo), serileştirme.
  • test/engine/: kural matrisi + senaryo + fuzz/değişmez + serde testleri (motor coverage ≥ %95).
  • Elle yazılmış 2 sabit test bölümü (test/fixtures/): mini bölüm (2 kategori) ve dolu bölüm (5 kategori, kilitli başlangıç sütunlu) — Faz 3'ün de geliştirme malzemesi.

3. Çekirdek Tasarım

3.1 Modeller

// lib/engine/models/
sealed class GameCard { final String id; }                    // id: "w:meyveler:3" | "c:meyveler"
class WordCard extends GameCard { final String word; final String categoryId; }
class CategoryCard extends GameCard { final String categoryId; final String name; final int totalInLevel; }

class ColumnPile {           // bir tableau sütunu
  final List<GameCard> faceDown;   // alt → üst
  final List<GameCard> faceUp;     // alt → üst; DEĞİŞMEZ: CategoryCard varsa yalnızca son elemandır
  bool get isLocked;               // faceUp.isNotEmpty && faceUp.last is CategoryCard
}

sealed class FoundationSlot {}
class EmptySlot extends FoundationSlot {}
class ActiveSlot extends FoundationSlot {
  final CategoryCard card; final List<WordCard> collected;   // kart kimlikleri korunur (undo/serde)
  bool get isComplete;             // collected.length == card.totalInLevel
}

class GameState {            // tamamen immutable; copyWith ile türetilir
  final List<ColumnPile> columns;  // v1: 5, motorda parametrik
  final List<FoundationSlot> slots;
  final List<GameCard> stock;     // üst = son eleman
  final List<GameCard> waste;
  final int movesLeft; final int completedCount;
  final GameStatus status;         // playing | won | lostOutOfMoves | lostDeadlock
  final LevelDef level;
}

3.2 Hamleler ve birimler

sealed class Move {}
class Draw extends Move {}
class Recycle extends Move {}
class Place extends Move { final UnitRef unit; final TargetRef target; }

sealed class UnitRef {}
class ColumnUnit extends UnitRef { final int column; final int startIndex; } // faceUp içi indeks
class WasteUnit extends UnitRef {}

sealed class TargetRef {}
class ColumnTarget extends TargetRef { final int column; }
class FoundationTarget extends TargetRef { final int slot; }

unitCards(state, unit) birimi çözümler ve tutulabilirlik kuralını uygular: U1 = startIndex'ten tepeye kesintisiz aynı kategori kelimeleri; U2 = aynı şey + tepede o kategorinin kartı; U3 = atık üstü tek kart. Aksi her şey V_NOT_A_UNIT.

3.4 Reducer ve olaylar

class ApplyResult { final GameState next; final List<GameEvent> events; }
apply(GameState, Move) → Result<ApplyResult, RuleViolation>

// GameEvent — UI animasyon kuyruğunun dili (Faz 3 bunları oynatır):
Drew(card) | Recycled(count) | UnitPlaced(unit, from, to)
| Flipped(column, card)                 // otomatik, bedava (§7)
| SlotActivated(slot, category, sweptWords)   // süpürmeyle gelen kelimeler dahil (§8.1)
| WordsCollected(slot, words)
| CategoryCompleted(slot, category)     // slot boşaldı (§8.3)
| GameWon | GameLost(reason)

Uygulama sırası: doğrula → hamleyi uygula → movesLeft-- → otomatik çevirme → tamamlama kontrolü → kazanma kontrolü → (playing ise) hamle-bitti / çıkmaz kontrolü → olay listesi.

3.5 Legal hamleler ve çıkmaz

legalMoves(state)   // tüm geçerli Place + Draw/Recycle; UI ve çözücü ortak kullanır
isDeadlocked(state) // spec §9.3: (1) hiçbir legal Place yok VE
                    // (2) stock ∪ waste'teki hiçbir kartın tahtada geçerli hedefi yok

3.6 Oturum, geri alma, serileştirme

  • GameSession: LevelDef + oynanmış List<Move> + durum yığını. play(move), undo() (son hamleyi düşürür; motor sınırsız destekler, kredi kontrolü Faz 4'ün işi). Kayıp durumundan undo() statüyü playing'e döndürür ("geri al & devam et", spec §9.4).
  • Kalıcı kayıt formatı = replay: {levelId, moves:[...]}. Yüklerken baştan oynatılır (motor deterministik). Küçük, sürüm-dayanıklı, spec K14 ile uyumlu. Anlık durum yığını yalnızca bellekte tutulur (O(1) undo).
  • serde.dart: LevelDef/Move/kayıt JSON gidiş-dönüşü. GameState için de tanılama amaçlı toJson (golden/debug).

5. Test Planı — kural matrisi eşlemesi

Her hücre en az bir pozitif ve (geçersizse) bir negatif test alır. Adlandırma: matrix_<birim>_<hedef>_<beklenti>.

Test grubuKapsanan durumlar
matrix_u1_*Kelime zinciri → dolu sütun (eşleşen ✓ / eşleşmeyen ✗), boş sütun ✓, kilitli sütun ✗, boş slot ✗, aktif slot (eşleşen ✓ tümü toplanır / eşleşmeyen ✗)
matrix_u2_*Süpürme birimi → eşleşen zincirli sütun ✓ (kilitlenir), boş sütun ✓ (kilitlenir), eşleşmeyen sütun ✗, kilitli ✗, boş slot ✓ (kelimeler toplanır: 0'lı ve n'li), aktif slot ✗
matrix_u3w_*Atıktan kelime → 5 hedef (U1 ile aynı beklentiler, tek kart)
matrix_u3c_*Atıktan kategori kartı → eşleşen sütun ✓ (kilit), eşleşmeyen ✗, boş sütun ✓ (kilit), boş slot ✓ (0/N), aktif slot ✗
scenario_*Süpürme (§8 örneği birebir: 7 hamlelik akış), tamamlama + slot boşalması + yeni kategoriye açılış, kilitli başlangıç sütunu, devir sıra korunumu (çek-çek-devir-çek aynı kartlar), hamle düşümleri (her eylem 1; otomatikler 0), kazanma (tahta boş), hamle bitince kayıp, çıkmaz kaybı
deadlock_*Pozitif: deste dolu ama hiçbir kart hiçbir yere oynanamıyor → çıkmaz. Negatif: tek legal hamle stokta gömülü kartın oynanabilirliği → çıkmaz değil. Sınır: yalnız faydasız ileri-geri hamle varken → çıkmaz DEĞİL (bilinçli, K13)
undo_* / serde_*Her hamle tipinde geri alma özdeşliği; kayıptan devam; replay determinizmi; JSON gidiş-dönüş
fuzz_invariantsKart korunumu (multiset), faceUp'ta kategori kartı hep sonda, kilitliye yerleştirme legalMoves'ta asla yok, collected ≤ total, movesLeft monoton azalır, statü geçişleri tekil

7. Bitti Tanımı (DoD)

  • Kural matrisi ve senaryo testlerinin tamamı yeşil; motor coverage ≥ %95 (M2).
  • Fuzz 1.000 oyun ihlalsiz; tüm motor test paketi < 60 sn.
  • Mimari koruma testi hâlâ yeşil (Flutter importu yok, dart:math Random yok).
  • Fixture bölümler çözüm replay'leriyle birlikte hazır (Faz 3 girdisi).

8. Riskler

  • Kural boşluğu keşfi: Testler yazılırken spec'te öngörülmeyen durum çıkarsa (örn. yeni kenar durumu) spec-önce prosedürü işletilir; karar kaydına eklenir.
  • Performans: legalMoves çözücünün sıcak döngüsü — immutable kopyalamada liste paylaşımı (unmodifiable görünümler) kullanılır; profil Faz 2'de doğrulanır.
