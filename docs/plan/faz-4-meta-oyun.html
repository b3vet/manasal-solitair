Faz 4 — Meta Oyun: İlerleme, Cüzdan, Öğretici

Büyüklük: L · Ön koşul: Faz 2 + Faz 3 · Durum: Planlandı · Dayanak: Spec v1.1 §9.4, §10, §13.1

1. Amaç

Tek elin etrafına oyunu inşa etmek: ekran akışı, sıralı bölüm ilerleyişi, kalıcı kayıt, geri alma cüzdanı + başarımlar, oyun-ortası devam etme ve rehberli öğretici. Faz sonunda uygulama "ilk açılıştan 100. bölüme" eksiksiz bir oyun döngüsü sunar (M5: dikey dilim).

2. Çıktılar

  • Ekran akışı: Splash → Ana Menü → Bölümler → Oyun (+ Ayarlar, Nasıl Oynanır).
  • Kalıcılık servisi (şema sürümlü) + otomatik kayıt/devam etme.
  • Cüzdan (kaynaklı işlem API'si) + başarım motoru + kutlama arayüzleri.
  • Kayıp ekranında "geri al & devam et" dahil tam geri alma entegrasyonu.
  • Bölüm 1–3 rehberli öğretici + "Nasıl Oynanır" sayfaları.

3. Kalıcı Veri Şeması (v1)

// shared_preferences üstünde JSON belgeleri; her belgede schemaVersion alanı
progress:      { schemaVersion: 1,
                 levels: { "12": { won: true, bestMovesLeft: 9 } },   // bestMovesLeft: K16 gereği saklanır
                 totalCategoriesCompleted: 37 }                        // Koleksiyoncu sayacı
wallet:        { schemaVersion: 1, balance: 3,
                 txLog: [ { t: "...", amount: +2, source: "achievement:first_win" } ] }  // son 50 işlem
achievements:  { schemaVersion: 1,
                 granted: ["welcome","first_win","efficient:12"],      // 'efficient' bölüm başına 1 kez
                 streakFirstTry: 2 }
settings:      { schemaVersion: 1, sound: true, haptics: true, reducedMotion: "system" }
resume:        { schemaVersion: 1, levelId: 12, moves: [...] }         // motor replay formatı (Faz 1)

Bozuk/eski veri: okuma hatasında belge yedeklenir (*_corrupt) ve temiz varsayılan kurulur; sürüm ileri taşıma (migration) çatısı ilk günden vardır (Faz 5 dayanıklılık testleri derinleştirir).

4. Görevler

IDGörevDetay ve teknik notlarKabul kriterleri
F4.1 Navigasyon ve ekran iskeletleri Yalın Navigator + tipli rota yardımcıları (go_router'a gerek yok — akış sığ). Splash (asset ön-yükleme + sürüm), Ana Menü (Oyna: kaldığı bölüme, Bölümler, Nasıl Oynanır, Ayarlar), yerleşimler tema tokenlarıyla. Akış uçtan uca gezilebilir; geri tuşu davranışları doğru (oyun ekranında onaylı çıkış).
F4.2 Kalıcılık servisi lib/persistence/store.dart: belge oku/yaz (JSON), şema sürüm kontrolü, migration kancası, bozuk veri kurtarma (§3). Tüm erişim Riverpod servis sağlayıcıları üzerinden; senkron önbellek + asenkron yazma (debounce 300ms). Birim testler: gidiş-dönüş, migration (v0→v1 örneği), bozuk JSON kurtarma.
F4.3 Cüzdan servisi API (spec §10.3): credit(amount, source) / tryDebit(reason) → bool; bakiye asla negatif olmaz; işlem günlüğü son 50 kayıt. source değerleri: achievement:*, ileride purchase:*, gift:*. Başlangıç bakiyesi 3 ("Hoş geldin", ilk açılışta bir kez — K17). Birim testler: negatif engeli, çift "hoş geldin" engeli, günlük sınırı.
F4.4 Başarım motoru Tetikleyiciler (K17 seti): first_win +2; efficient (harcanan ≤ limit×0.6; bölüm başına 1 kez) +1; streak_3 (art arda 3 bölüm ilk denemede — deneme sayacı bölüm başına tutulur) +1; collector_25/50/100 +1/+2/+3. Değerler tek sabit dosyada (achievements_config.dart). Kazanım anı: kazanma diyaloğunda "+N geri alma" rozeti + kısa kutlama. Her tetikleyicinin birim testi (sınır durumlarıyla: tekrar kazanma, seri bozulması); kazanma diyaloğu doğru toplamı gösteriyor.
F4.5 Geri alma entegrasyonu HUD düğmesi bakiyeyi gösterir (↩ 3); basışta tryDebit("undo") → başarılıysa session.undo() (hamle sayacı geri yüklenir — motor Faz 1'de hazır). Bakiye 0: düğme soluk; dokununca "kredi kazanma yolları" alt sayfası (başarım listesi + ilerlemeler). Kayıp diyaloğunda "Geri al ve devam et" (1 kredi): statü playing'e döner, oyun sürer (spec §9.4). Art arda basış: her adım 1 kredi. Widget testleri: bakiye düşümü, 0 bakiye akışı, kayıptan devam; undo sonrası HUD tutarlı.
F4.6 Devam etme (resume) Her hamlede debounce'lu otomatik kayıt (replay formatı). Uygulama açılışında resume varsa "Oyna" doğrudan kaldığı yere götürür (toast: "Kaldığın yerden devam ediyorsun"). Kazanmada kayıt silinir; kayıpta silinmez — "geri al & devam et" uygulama yeniden açılışında da mümkün olsun (kayıp durumu replay'den yeniden kurulur). Bölümü elle yeniden başlatmak kaydı sıfırlar. Testler: kapat-aç → aynı durum (hamle sayısı, kilit, slotlar); kayıpta aç → kayıp diyaloğu + kurtarma çalışıyor.
F4.7 Bölüm listesi Sayfalı ızgara (20'li gruplar); durumlar: kilitli / oynanabilir (sıradaki) / tamamlandı (✓). Sıralı ilerleyiş (K/spec §13.1): N tamamlanmadan N+1 kapalı; tamamlananlar tekrar oynanabilir (tekrar kazanım kredi/başarım saymaz — yalnız bestMovesLeft güncellenir). Üstte toplam ilerleme: bölüm + kategori sayacı (Koleksiyoncu ilerlemesi görünür). Widget testleri: kilit mantığı, tekrar oynama etiketleri; 100 bölümde akıcı kaydırma.
F4.8 Öğretici Script motoru: adım = { vurgu (BoardMetrics dikdörtgeni + karartma maskesi), metin balonu, beklenen eylem (yalnız o eyleme izin veren girdi kapısı) }. Bölüm 1–3 senaryoları (Faz 2'de sabitlenen tohumlarla yazılır — dağılım bilindiği için adımlar deterministik): B1 çekme + kelime dizme + kategori açma + toplama; B2 süpürme + kilit; B3 devir + çıkmaz uyarısı + geri alma tanıtımı. "Atla" her an var; tamamlanma bayrağı kalıcı. Ayrıca menüden "Nasıl Oynanır": 5 resimli statik sayfa (kurallar özeti, spec §13.1). Script motoru birim testli (kapı ihlali ilerletmez); 3 bölüm uçtan uca rehberle oynanıyor; KULLANICI ONAYI: öğretici akışı.
F4.9 Ayarlar ve hakkında Ses/haptik anahtarları (Faz 5'te işlevlenir, anahtar bugünden kalıcı), animasyon azaltma (sistem/açık/kapalı), öğreticiyi sıfırla, "Nasıl Oynanır" linki, sürüm/lisans bilgisi. Ayarlar kalıcı; reduce-motion seçimi uçuş koordinatörüne anında yansıyor.

6. Bitti Tanımı (DoD)

  • İlk açılış → öğretici → bölüm ilerleyişi → kayıp/kurtarma → kazanç/kredi döngüsü uçtan uca çalışıyor (M5).
  • Kalıcılık: kapat-aç senaryoları (oyun ortası, kayıp anı, bölüm listesi) testli.
  • Cüzdan/başarım birim testleri yeşil; değerler tek config dosyasında.
  • Öğretici kullanıcı onaylı; web dağıtımı güncel.

7. Riskler

  • Öğretici-bölüm bağı: B1–3 dağılımı değişirse script kırılır → bu bölümlerin tohumları üretim hattında "dondurulmuş" işaretlidir; yeniden üretimde değişmezlik testi vardır (F2.7).
  • Kayıpta devam kurtarması: replay'den kayıp durumunun yeniden kurulması motorun deterministik statü geçişlerine dayanır — Faz 1 testleri bunu garanti eder; ek entegrasyon testi burada yazılır.
  • Ekonomi dengesi: değerler config'te; Faz 5 QA'sında gerçek oynanış verisiyle (kaç bölümde kaç kredi birikti) gözden geçirilip kullanıcıyla kalibre edilir.
