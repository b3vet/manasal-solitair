4. Çalışma Pratiği

  • Dal: Tüm geliştirme claude/word-solitaire-flutter-3ru19g dalında yapılır ve oraya push'lanır.
  • Spec-önce prosedürü: Geliştirme sırasında bir kural belirsizliği/ihtiyacı çıkarsa önce spec güncellenir (sürüm artar, karar kaydına satır eklenir), sonra kod yazılır. Kod hiçbir zaman spec'ten sapan davranış içeremez.
  • Test-önce motor: Faz 1'de kural testleri gerçeklemeden önce/birlikte yazılır; kural matrisi test adlarıyla birebir eşlenir.
  • Faz kapanışı: Faz bitince ilgili plan dokümanının durumu güncellenir, DoD listesi işaretlenir, kısa kapanış notu eklenir.
  • Her faz sonunda çalışan ürün: CI her push'ta analiz + test + web build üretir; Faz 3'ten itibaren otomatik web test dağıtımı (GitHub Pages) ile telefonda denenebilir.
Ortam notu: Bu uzak geliştirme ortamında Flutter SDK kurulu değildir (kontrol edildi). Faz 0'ın ilk görevi, sabitlenmiş sürümlü bir scripts/setup.sh ile SDK kurulumunu tek komuta indirmek ve aynı kurulumu CI'da da kullanmaktır. SDK indirilemezse bile motor + üretici katmanları saf Dart olduğu için dart test ile ilerlenebilir; UI fazları web build gerektirir.
